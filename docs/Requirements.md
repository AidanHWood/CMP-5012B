------- Functional Requirements -------
1. Food and Calorie Tracking<br>
        The system will allow users to log meals and snacks with nutritional content.
2. Workout Tracker
        Users will be allowed to log workouts including type, duration, intensity and calories burned.
3. Progress Visualisation
        The system will display the user's progress over time using charts and summaries.
4. User Accounts
        The system will allow users to log in, create an account and store personal data securely.
5. Reminders and Notifications
        The system will send notifications to users (if enabled) to encourage working towards goals.

------- Non-Functional Requirements -------
1. Performance
        On a standard mobile connection, the application should load within a respectable time (e.g. 3s)
3. Usability
        The application should minimise the number of interactions required to log a meal, workout, goal, etc.
4. Privacy and Compliance
        The system will adhere to GDPR and allow users to delete the information we store on them.
5. Maintainability
        The system should be modular to allow for new features/changes to be made quickly.
6. Accessibility
        The application should be built to fit the dimensions of most/all phone models to allow a range of users.
